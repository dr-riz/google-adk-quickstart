# Google ADK Starter Project

This repository is a minimal Google ADK workspace centered on a single agent package, `my_agent`.

## Project Analysis

Current structure (relevant files):

- `.gitignore`
- `requirements.txt`
- `my_agent/agent.py`
- `my_agent/__init__.py`
- `.adk/` (runtime artifacts)
- `my_agent/.adk/` (agent runtime state)
- `my_agent/.env` (local environment variables)

What this project currently does:

- Defines one root LLM agent in `my_agent/agent.py`.
- Uses Gemini model `gemini-2.5-flash`.
- Is intended to run through ADK CLI (`adk run` or `adk web`).

Notes about runtime folders:

- `.adk/` and `my_agent/.adk/` are generated by ADK at runtime.
- They may include local sessions and artifacts (for example `session.db`).

## Prerequisites

- Python 3.13 (the workspace currently uses a local `.venv`)
- A Gemini API key
- ADK dependencies installed (already captured in `requirements.txt`)

## Setup

1. Create and activate a virtual environment (if you do not already have one):

   python3 -m venv .venv
   source .venv/bin/activate

2. Install dependencies:

   pip install -r requirements.txt

3. Provide your API key.

   Recommended: place it in `my_agent/.env`:

   GOOGLE_API_KEY=your_api_key_here

   Or export it in your shell:

   export GOOGLE_API_KEY=your_api_key_here

## Run The Agent

### Option A: Terminal chat

Run the agent package directly:

adk run my_agent

### Option B: Web UI

Start ADK web server:

adk web --port 8000

Then open the local URL shown in terminal.

## Agent Definition

`my_agent/agent.py` defines:

- `root_agent` as an ADK `Agent`
- model: `gemini-2.5-flash`
- instruction: "Answer user questions to the best of your knowledge"

## Troubleshooting

### `ModuleNotFoundError: No module named 'adk'`

Use Google ADK import paths and CLI from your venv:

- Python imports should come from `google.adk...`
- Run commands with the environment active (`source .venv/bin/activate`)

### API key errors

If you see an API key error, verify:

- `GOOGLE_API_KEY` is set
- You are using the same shell/environment where the key is defined

### Ctrl+C stops `adk web` with exit code 130

This is expected behavior for manual interrupt.

## Dependency Notes

`requirements.txt` was generated from a full environment freeze. It is valid for reproducing the exact environment, but it includes many transitive packages.
